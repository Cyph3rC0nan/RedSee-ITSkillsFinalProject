RedSees Marketplace — Ground-Truth Vulnerability Map
=====================================================

What this is: the authoritative answer key for the RedSees Marketplace demo lab
target. This file is BOTH human documentation and a scanner-evaluation fixture —
every intentionally planted vulnerability in the demo-target/ companion service is
recorded here, one delimited block per sink, in a consistent key: value shape so a
human or a future agent session can parse it without ambiguity. Anything NOT listed
in this file is not an intentionally planted vuln for this demo pass (XSS-only scope
— no SQLi/IDOR/auth issues were added this pass).

Base URL / hostname: http://redsees.com:3000/market/
Themed Juice Shop (branding only, no planted vulns of its own beyond stock Juice
Shop challenges): http://redsees.com:3000/ (NODE_CONFIG_ENV=redsees)

Port note (2026-07-14): the sink service and Juice Shop now sit behind ONE public
port via demo-target/gateway/, a reverse proxy (/market/* -> the sink service,
everything else -> Juice Shop). The sink service's own code (demo-target/
marketplace_sinks.py) is unchanged and still internally listens on :8081 — only
the externally-reachable port changed, from :8081 directly to :3000 via the
gateway. All `url:` fields below reflect the current, gateway-fronted address.

Authorization: lab-only target, reachable from the RedSee sandbox exclusively,
never exposed publicly. All proofs are benign (`alert(1)`-style). No real user data.

Verified 2026-07-12 (pre-gateway, direct :8081): live `dalfox url` run (v2.13.0,
via the redsee-sandbox image) against all three sinks each produced a
`[POC][V][GET][...]` line, e.g.
`[POC][V][GET][inHTML-URL] http://localhost:8081/market/search?q=test%3E...`
